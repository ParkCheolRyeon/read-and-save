# 챕터 리더 재구성 Design

## 목표

로그인 이후 기존 카드형 목록 UI는 유지한 채, 기존 샘플 책 데이터를 10개 면접 챕터로 교체한다. 각 챕터는 개별 마크다운 파일로 제공하고 기존 리더 화면에서 그대로 읽을 수 있게 한다. 모바일에서 리더를 끝까지 스크롤했을 때 위로 튀는 현상은 읽기 UI를 유지하면서 제거한다.

## 범위

- `public/books`의 기존 샘플 마크다운과 manifest를 10개 챕터 데이터로 교체
- 제목을 정확히 노출할 수 있도록 manifest를 메타데이터 형태로 확장
- 기존 `BookList` UI는 재사용
- 기존 `Reader` UI는 유지하되 모바일 스크롤 버그를 유발하는 강제 snap 동작 제거

## 데이터 구조

- `public/books/manifest.json`은 각 챕터의 `id`, `filename`, `title`을 담는 배열로 변경
- `public/books/*.md`는 챕터별 본문 저장소 역할만 담당
- `src/lib/books.ts`는 새 manifest 형태를 읽어 `BookEntry[]`를 반환

## UI/동작

- 로그인 화면은 변경 없음
- 로그인 후 목록에는 10개 챕터 모두 노출
- 카드 모양, 헤더, 로그아웃 버튼, 리더 레이아웃은 유지
- 리더는 긴 단일 문서를 자연스럽게 스크롤하도록 하고, 기존 맨 위로 버튼은 유지

## 버그 수정 방향

- 현재 리더는 긴 문서 하나를 렌더링하면서 `scrollSnapType: "y mandatory"`를 적용하고 있어 모바일에서 끝 지점 근처에 비정상적인 강제 정렬이 발생할 가능성이 큼
- 읽기 페이지에서는 페이지형 스냅보다 자연 스크롤이 더 맞으므로 스냅을 제거하고 문서 높이 계산도 단순화

## 검증

- 정적 검증: `eslint`
- 런타임 검증: 로컬 서버에서 로그인 후 목록/리더 진입 확인
- 모바일 검증: 모바일 뷰포트에서 하단까지 스크롤 시 위로 튀지 않는지 확인

## 비고

- 이 문서는 로컬 정책상 별도 승인 없이 커밋하지 않는다.
