# 전체 챕터 누락 문항 재점검

이 문서는 처음 제공한 **200개 원문 문항** 기준으로,
현재까지 만든 **플로우형 챕터 1~11**에 무엇이

- 완전 반영되었는지
- 부분 반영 상태인지
- 아직 누락되었는지

를 다시 점검한 결과다.

---

## 재점검 기준

### 완전 반영

- 현재 플로우 문서 안에 **문항의 중심 주제가 독립 섹션**으로 존재함
- 면접 답변용 압축까지 어느 정도 연결 가능함

### 부분 반영

- 흐름 안에서 언급되거나 관련 개념은 들어갔지만
- **원문 문항 수준의 독립 질문/해설**로는 아직 부족함

### 누락

- 현재 플로우형 문서 안에
- **직접 대응되는 독립 섹션이 아직 없음**

- 13. `async/await`
- 14. `Promise.all / allSettled / race / any`
- 15. 디바운스와 스로틀

## 부분 반영

- 17. 브라우저 렌더링 과정에서 JavaScript 영향
  - 챕터 8의 렌더 차단 리소스, 메인 스레드 점유, INP 쪽에 연결은 됐지만
  - **JS 관점 독립 문항**으로는 아직 없음
- 18. 브라우저 메모리 누수
  - 클로저 오용, 오래된 참조, 메모리 유지와는 연결됐지만
  - **브라우저 이벤트 리스너 / DOM 참조 / 타이머 / detached node** 중심 설명이 별도 없음

## 누락

- 8. 화살표 함수 vs 일반 함수
- 9. 프로토타입 체인 / 클래스 문법 내부 동작

- 16. 이벤트 버블링 / 캡처링 / 이벤트 위임

### 요약

JS는 **초반 실행 모델 + 비동기 모델**은 꽤 잘 반영됐고,  
반대로 **언어 객체 모델(this/prototype/comparison/event propagation/collection)** 축이 아직 많이 남아 있다.
